# Claude Code - 友達のウェブサイト作成プロジェクト設定

## プロジェクト概要
このリポジトリは、友達から依頼されたウェブサイトを管理するためのプロジェクトです。

## ディレクトリ構造
```
website/
├── CLAUDE.md          # このファイル（Claude Codeの設定とルール）
├── README.md          # プロジェクトの説明
└── [友達の名前]/      # 各友達のウェブサイト用ディレクトリ
    ├── index.html     # メインHTMLファイル
    ├── style.css      # カスタムCSS（必要に応じて）
    └── assets/        # 画像やその他のリソース（必要に応じて）
```

## URL構造
各ウェブサイトは以下の形式でアクセス可能：
- `https://yourdomai.com/[友達の名前]/`

## ウェブサイト作成ワークフロー

### 1. 依頼受付時の確認事項
新しいウェブサイトの依頼を受けたら、以下のカテゴリーに沿って必要な情報を確認します。
すべての項目が必須ではありませんが、より良いウェブサイトを作成するために可能な限り収集することを推奨します。

#### 【必須】基本情報
1. **サイトオーナー**
   - 友達の名前（ディレクトリ名とURLに使用）
   - ローマ字またはURLに適した形式
   - 連絡先情報（メール、電話など）

2. **ウェブサイトの目的・用途**
   - 個人ポートフォリオ
   - ビジネス紹介
   - ランディングページ
   - Eコマース
   - ブログ/メディアサイト
   - 趣味のサイト
   - イベント告知
   - SaaS製品紹介
   - その他

3. **ウェブサイトの主な目標**
   - 何を達成したいか（例：問い合わせ獲得、商品販売、情報提供など）
   - 成功をどう測定するか（例：月間訪問者数、問い合わせ件数など）

#### 【必須】ターゲットオーディエンス
4. **訪問者像**
   - ターゲットとなる主なオーディエンスは誰か
   - 年齢層、職業、興味関心など
   - ユーザーが求める重要な情報は何か
   - ユーザーからの想定される質問

#### 【必須】デザイン・ブランディング
5. **ビジュアル要素**
   - ロゴの有無（ある場合は提供を依頼）
   - カラースキーム（好きな色、ブランドカラー、避けたい色）
   - ブランドガイドラインの有無
   - 標準フォントの有無

6. **デザイン嗜好**
   - 参考になるウェブサイトの例（URL）
   - 好きなデザインスタイル（シンプル、モダン、クラシック、ミニマル など）
   - 既存サイトがある場合、好きな点・嫌いな点
   - 特別な機能やエフェクトの要望
   - 避けたいデザイン要素

#### 【必須】コンテンツ詳細
7. **サイト構成**
   - 予想されるページ数
   - 必要なセクション構成（Hero, About, Services, Portfolio, Contact など）
   - ナビゲーション構造

8. **コンテンツ素材**
   - 掲載するテキスト内容（完成版か草案か）
   - コピーライティングサービスが必要か
   - 画像の有無と種類（写真、イラスト、アイコンなど）
   - 画像提供の有無（ない場合はストック写真使用）
   - 動画コンテンツの有無

9. **連絡先・SNS情報**
   - 掲載する連絡先情報（住所、電話、メールなど）
   - SNSアカウント（Instagram, Twitter/X, Facebook, LinkedIn, GitHub など）
   - SNS統合の必要性（埋め込みフィード、シェアボタンなど）

#### 【推奨】機能要件
10. **必要な機能**
    - お問い合わせフォーム
    - ニュースレター登録
    - ブログ機能
    - 検索機能
    - 多言語対応
    - ダークモード
    - アニメーション・インタラクション
    - オンラインストア（Eコマース機能）
    - 予約システム
    - ログイン/会員機能
    - その他の特殊機能

11. **Eコマース詳細**（必要な場合）
    - 商品数と種類
    - 配送方法（国内/国際/店舗ピックアップ）
    - 決済方法の希望
    - 商品画像と説明文の有無

12. **アナリティクス・統合**
    - Google Analyticsの設定が必要か
    - 既存のアクセス解析データの有無
    - 他ツールとの連携（CRM、メールマーケティングツールなど）

#### 【推奨】競合・差別化
13. **競合分析**
    - 主な競合企業・サイトは誰か（URL）
    - 競合との差別化ポイント
    - 競合が優れている点（参考にすべき点）

#### 【推奨】技術的要件
14. **既存の技術資産**
    - 既存ウェブサイトの有無とURL
    - 使用しているプラットフォーム（WordPress, Webflow など）
    - ドメインの有無とレジストラ
    - ウェブホスティングの有無とホスト企業
    - 技術的な制約や要件

#### 【推奨】プロジェクト管理
15. **スケジュール・予算**
    - 希望する公開時期・期限
    - 予算の目安（ある場合）
    - 急ぎの理由（期限がある場合）

16. **メンテナンス・更新**
    - サイト公開後に更新する必要があるか
    - 更新責任者は誰か
    - 更新方法の研修が必要か
    - サイトにアクセスする必要がある人（複数の管理者）

#### 【オプション】その他
17. **追加情報**
    - ユーザー属性（開発者、起業家、デザイナー、マーケターなど）
    - このプロジェクトをどこで知ったか
    - チームメンバーや協力者の有無
    - 特別な要望や懸念事項
    - ビジネスのミッションステートメント
    - 企業文化や価値観

### 2. 作成プロセス
1. **【必須】カテゴリーの情報を収集**
   - 最低限、基本情報、ターゲットオーディエンス、デザイン・ブランディング、コンテンツ詳細の情報を収集

2. **【推奨】カテゴリーの情報を収集**
   - より良いウェブサイトを作成するため、可能な限り推奨項目も収集

3. **不足情報の確認**
   - 重要な情報が不足している場合は追加で質問
   - 自然な会話形式で段階的に情報を引き出す

4. **実装開始**
   - 最低限必要な情報が揃った段階で以下を実行：
     - `/website/[友達の名前]/` ディレクトリを作成
     - `index.html` を作成
     - 必要に応じて `style.css` を作成
     - 必要に応じて `assets/` ディレクトリを作成

### 3. 技術仕様
- HTML5を使用
- レスポンシブデザイン対応
- 必要に応じてカスタムCSSを作成
- 参考サイトがある場合は、そのデザインスタイルを参考に実装
- モダンなWebデザインのベストプラクティスに従う

### 4. Git管理フロー
1. ウェブサイト作成時は `claude/` で始まる作業ブランチで作業
2. 作成完了後、自動的に `main` ブランチにマージ
3. ユーザーの許可を待たずに自動マージを実行
4. すべての変更はコミットメッセージと共に記録

## 注意事項
- 各友達のウェブサイトは独立したディレクトリで管理
- ディレクトリ名はURLに使用されるため、半角英数字とハイフンのみ使用
- すべての変更はGitで管理し、適切にコミット・プッシュする
- 作業完了後は自動的にmainブランチにマージ
